# /log

Append a timestamped entry to `.claude/journal.md` in the current project.

## Entry types

| Type | Use for |
|---|---|
| `decision` | A load-bearing choice that shapes the codebase or product. Future contributors need to know why. |
| `warning` | A landmine, known gap, or thing that will bite someone if they don't know about it. |
| `discovery` | Something learned about the domain, the codebase, or the user. |
| `state` | A snapshot of where the project stands right now. |
| `todo` | A deferred item that doesn't have a ticket yet. |
| `note` | Anything else. Default if no type given. |

## How to invoke

```
/log decision "Price is never AI-inferred — non-negotiable"
/log warning "CSV enum strings not yet validated against real Whatnot import"
/log state "Phase 1 source files written, Expo scaffold not yet run"
/log discovery "Whatnot requires integer prices — no cents in CSV"
/log todo "Add requestMediaLibraryPermissionsAsync before image picker"
/log "Just a quick note without a type"
```

Arguments are flexible. The type can appear anywhere in the invocation or be omitted entirely.

## Steps

### 1. Parse the invocation

Extract:
- **type**: one of `decision`, `warning`, `discovery`, `state`, `todo`, `note`. Default: `note`.
- **message**: the rest of the text. Preserve the user's exact wording — do not paraphrase.

If the user provides additional context beyond what fits on one line, include it.

### 2. Find the project root

Run:
```bash
git rev-parse --show-toplevel 2>/dev/null || pwd
```

Use the result as `PROJECT_ROOT`.

### 3. Ensure .claude directory exists

```bash
mkdir -p "{PROJECT_ROOT}/.claude"
```

### 4. Format the entry

Use this exact format:

```
---
{YYYY-MM-DD HH:MM} · {TYPE_UPPERCASE}
{message}

```

Example:
```
---
2026-03-19 14:32 · DECISION
Price is never AI-inferred — non-negotiable. Sellers price their own inventory. Not even a range. Not even a suggestion in the description field.

```

Get the current timestamp with:
```bash
date '+%Y-%m-%d %H:%M'
```

### 5. Append to journal

Append the formatted entry to `{PROJECT_ROOT}/.claude/journal.md`.

If `journal.md` does not exist, create it with this header first:

```markdown
# Project Journal

Append-only log of decisions, warnings, discoveries, and state snapshots.
Generated by /log. Synthesized by /brief. Read by /orient.

```

Then append the entry.

### 6. Confirm

Tell the user exactly one line:
```
Logged [TYPE]: "{first 60 chars of message}"
```

No other output. Keep it frictionless.
